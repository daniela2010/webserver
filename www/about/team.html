Project Division & Milestones

Our development lifecycle was engineered in parallel modules, ensuring high separation of concerns across the network and application-level components.

Part A: Socket Operations

Handles the initial TCP connection workflow, server socket initialization, incoming connection acceptance, and multi-threaded client handling.

Part B: HTTP Stream Parser

Parses incoming HTTP requests, extracts methods, requested URLs, and HTTP versions, then builds valid HTTP responses.

Part C: Resource & Security Manager

Resolves static resources, validates requested paths, detects MIME types, prevents directory traversal attacks, and enforces access restrictions.
