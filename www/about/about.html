Technical Specifications & Features

Instead of deploying our site on pre-built enterprise software like Apache or Nginx, we manually implemented our own engine to handle client-server paradigms natively.

Implemented Server Mechanisms

  • Socket Management: Opens a strict ServerSocket listening to incoming ports (e.g., Port 8080 or Port 80).
  • MIME-Type Resolution: Evaluates request sub-strings to correctly bind header attributes such as Content-Type: text/css.
  • Security Sanitization: Inspects relative paths to intercept ../ malicious attempts, actively deflecting Directory Traversal breaches.
  • Stateless State Tracking: Employs structured layouts ready for expansion towards HTTP Cookies and user session parameters.

HTTP Response Validation

The backend checks whether resource structures match host directives, delivering robust 200 OK payloads, or generating programmatic 404 Not Found and 403 Forbidden errors gracefully.
